iot.agile.Device

Properties

  • String Id The unique device id in the gateway

  • String Name The device name

  • StatusType Status Indicate the current device status

  • KeyValue Configuration User configurations storage

  • Object* Profile Contains protocol specific provided information on device in order to handle at Protocol level the specific implementation

  • Number LastUpdate A UNIX timestamp to indicate the last data update received by the Device

  • Object* Data Retain the most recent updates of a sensor or data stream, received asynchronously from Subscribe call

  • Protocol Device-specific communication Protocol instance. Available to access protocol specific methods and properties

Methods

  • Object Execute( String op , Object* payload ) Execute an operation on the device triggering the underline Protocol implementation

  • Object* Read( String sensorName )

  • Boolean Write( String sensorName , Object* data )

  • Boolean Subscribe( String sensorName ) Enable subscription to a specific sensor or data stream, triggering the underlying Protocol implementation if supported.

ChannelTypeEnum : Enum

    DeviceStreamChannel : Object A channel contains a single measurement value of a sensor

    name : String

    type : ChannelTypeEnum

    DeviceStream : Object A stream contains a set of measurement values for a sensor

    name : String

    channels : Array[DeviceStreamChannel]

    __arrayType : DeviceStreamChannel

    DeviceDefinition : Object A device definition that contains informations describing the current device

    id : String

    name : String

    description : String

    properties : Object*

    streams : Array[DeviceStream]

    __arrayType : DeviceStream

    ProtocolDevice : Array List all devices a Protocol knows about. The list is updated by the Discover method. Devices may be provisioned programmatically during the initialization phase.

    type : Object

    fields :

    ProtocolStatus : Enum Identify a status of the Protocol, indicating if operational (AVAILABLE) or if for some reason further user-side checks are required.

    • AVAILABLE
    • NOT_CONFIGURED
    • UNAVAILABLE
    • DISABLED
    • FATAL

    ProtocolProfile : Object* A Protocol set of settings that translate human readable properties to actual Protocol implementation details. For example, the user request the temperature value over a BLE connected device and the Protocol receive all the specific informations (like services and characteristics ids) to enable the data reading.

    ProtocolOverview : Object

    name : String

    id : String

    status : ProtocolStatus

    DriverStatus : Enum

    • ACTIVE
    • NOT_CONFIGURED
    • ERROR
    • FAILURE

    RecordObject : Object

    DeviceId : String

    StreamId : String

    ChannelId : String

    LastUpdate : Number A UNIX timestamp rapresenting the last update of a record

    Channels : KeyValue A list of key/value matching those indicated in the DeviceDefinition

    KeyValue : Object* A list of key/value pairs

    BaseQuery : Object

    Criteria : Object* A key-value series of field-value to query for. - Use `*` to indicate a fuzzy search - Use an Array to indicate an IN like query

    OrderBy : Object* A key-value series of field-value to order the result for.

    Limit : Number The limit of records founds to return

    Offset : Number The position for the first record to take from the list of results
